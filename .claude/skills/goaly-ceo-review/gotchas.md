# CEO Review Gotchas

Failure modes specific to the ceo-review skill. Read before every run.

- **Read-only skill — never edit files directly.** This skill reviews and challenges plans. It never creates tasks, edits notion-mirror files, or writes code. All outputs are advisory text presented to [Owner].
- **Challenge scope creep and over-commitment.** [Owner]'s instinct is to say yes and expand scope. Your job is to push back — ask "is this [Owner]'s problem to solve?" and "what's being sacrificed?"
- **Check ikigai alignment before endorsing new commitments.** Score every plan against Love/Good At/Needs/Paid For. If below 2/4, do not proceed without explicit confirmation. See `memory/ikigai.md` for full dimensions.
- **Flag single-source revenue risk.** If a plan deepens dependency on one client without diversification, flag it. [Owner]'s principle: 3-4 revenue sources minimum, single-source = unacceptable risk.
- **Three modes have distinct postures — don't blend them.** SCOPE EXPANSION (dream big), HOLD SCOPE (maximum rigor), SCOPE REDUCTION (strip to essentials). Each has its own checklist and output format. Mixing postures produces incoherent advice.
- **Handoff to /brainstorm or /plan, never implement directly.** The recommended next step is always a skill invocation or a reject. Never produce implementation plans, code, or file edits.
